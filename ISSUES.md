# Common Agent Traps

- A stale ignored `port/hello-world/` directory can survive on disk after the real Phase 1 workspace is moved to `port/`. If Phase 1 docs claim the placeholder was replaced but the directory still exists locally, delete it instead of preserving the obsolete layout.
- Raw `SCENE.HQR` entry bytes are not the same thing as the runtime scene payload. The classic loader reads an HQR `COMPRESSED_HEADER` first and may LZ-decompress the entry before parsing scene fields, so typed decoders must decode the resource block before interpreting world/object/zone data.
- Scene numbering is easy to misread. The classic loader calls `Load_HQR(..., numscene + 1)` because `SCENE.HQR[0]` is reserved for `SizeCube.MAX`, while the current `inspect-scene` command accepts the raw one-based HQR entry index. Keep semantic scene numbers and HQR entry numbers separate, and do not revive the old `SCENE.HQR[4]` exterior misclassification just because that entry decodes cleanly.
- The raw-to-loader mapping has two offsets in this repo, not one. Classic code uses raw HQR slot `0` for `SizeCube.MAX`, while the Zig tooling exposes physical entries starting at `1`, so `SCENE.HQR[2]` is classic loader scene `0` and `classic_loader_scene_number` must use `entry_index - 2` for real scenes.
- The MBN `SCENE.HQR` lists use type markers that matter. In post `177798`, `~1` means isometric room data and `~2` means outside scene data; if you pick a golden exterior target from that list, keep the type marker with the descriptor instead of copying only the human-readable scene name.
- The checked-in memory files can lag behind the actual worktree state. As of 2026-03-19, `docs/codex_memory/current_focus.md` and `docs/codex_memory/handoff.md` still describe the repo as intentionally dirty in corpus/reference areas, but `git status --short` is clean. Verify the real worktree before treating the dirty-worktree warning as current fact.
- `zig build test` is currently a real-asset regression gate, not a hermetic unit-test-only pass. The scene tests resolve the canonical extracted asset tree by default, and `port/build.zig` hard-fails if the repo-local SDL2 install is missing, so do not assume the test step can run on a stripped-down workspace.
- `SCENE.HQR` giver zones do not encode one fixed bonus kind at load time. `Info0` is a bonus-flag mask that `WhichBonus()` resolves later from runtime state, so a typed decoder should expose the allowed bonus kinds or preserve the raw mask instead of inventing a singular `bonus_kind`.
- `COMMON.H` plus `GERETRAK.CPP` is not a complete track-opcode layout oracle on its own. Real scene `44` hero-track bytes decode repeated `TM_REM` opcodes even though `DoTrack` does not spell out a `TM_REM` switch case, and the checked-in runtime source still leaves `TM_NOP`, `TM_DO`, and `TM_AFF_TIMER` without explicit operand coverage there. Treat the structural decoder as the current byte-level truth and do not overclaim stronger semantics for those opcodes without new evidence.
